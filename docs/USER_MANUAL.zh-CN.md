# BlockForge 使用手册

BlockForge 是一个 Minecraft 风格 voxel 建筑蓝图生成器。你可以选择内置建筑模板，在网页中查看 3D 方块模型，并导出 JSON 或 Minecraft `.mcfunction` 文件。

## 1. 启动项目

在项目根目录运行：

```bash
pnpm install
pnpm dev
```

启动成功后，浏览器访问：

```text
http://localhost:3000
```

如果 `3000` 端口被占用，Next.js 可能会自动使用其他端口，例如：

```text
http://localhost:3001
```

以终端输出的地址为准。

## 2. 页面区域说明

首页主要分为三个区域：

- 顶部标题区：展示项目名称 `BlockForge` 和项目简介。
- 左侧控制面板：输入 prompt、选择 preset、导出文件。
- 右侧预览区域：展示当前选中建筑的 3D voxel 模型和模型信息。

在移动端，页面会自动变成上下排列。

## 3. 选择建筑模板

当前版本内置 5 个 preset：

- Medieval Tower：中世纪塔楼
- Small Cottage：小木屋
- Dungeon Entrance：地牢入口
- Stone Bridge：石桥
- Pixel Statue：像素雕像

点击任意 preset 后：

- 选中卡片会高亮。
- 右侧 3D 预览会切换到对应模型。
- 模型信息会同步更新，包括尺寸、方块数量和方块类型。
- 导出按钮会导出当前选中的模型。

## 4. 使用 Prompt 输入框

你可以在输入框里输入建筑描述，例如：

```text
a medieval tower with glass windows
```

点击 `Generate Blueprint` 后，页面会显示：

```text
Blueprint generated from prompt: ...
```

注意：v0.1 版本暂未接入真实 AI 生成逻辑。Prompt 当前只用于展示交互状态，不会生成新的 voxel 模型。真实 prompt-to-structure 生成器在 Roadmap 中。

## 5. 查看 3D 预览

3D 预览区域会显示当前选中的 `VoxelModel`。

你可以：

- 鼠标左键拖动：旋转模型
- 鼠标滚轮：缩放模型
- 鼠标右键或触控板拖动：平移视角

预览区包含：

- 真实 voxel 方块渲染
- 不同 block 类型对应不同颜色
- 半透明玻璃和水
- 地面网格
- 基础光照

## 6. 查看模型信息

预览区域会显示：

- 模型名称
- 模型描述
- 模型尺寸，例如 `11 x 16 x 11`
- 方块数量，例如 `656`
- 当前 prompt 状态
- 使用到的 block 类型列表

这些信息来自当前真实 `VoxelModel`。

## 7. 导出 JSON

点击：

```text
Export JSON
```

会下载当前模型的 JSON 文件。

JSON 内容包含：

```ts
{
  id: string;
  name: string;
  description: string;
  size: {
    width: number;
    height: number;
    depth: number;
  };
  blocks: {
    x: number;
    y: number;
    z: number;
    block: string;
  }[];
}
```

JSON 适合用于：

- 调试 voxel 数据
- 后续接入 AI 生成器
- 二次开发
- 自定义导出格式

## 8. 导出 .mcfunction

点击：

```text
Export .mcfunction
```

会下载 Minecraft Java Edition 可用的 `.mcfunction` 文件。

文件内容由 `setblock` 命令组成，例如：

```mcfunction
# Generated by BlockForge
# Model: Medieval Tower
# Size: 11 x 16 x 11
# Blocks: 656
setblock ~1 ~0 ~1 minecraft:cobblestone replace
setblock ~2 ~0 ~1 minecraft:cobblestone replace
```

说明：

- `.mcfunction` 文件中不会包含 `/setblock` 斜杠。
- 默认使用相对坐标，例如 `~`、`~1`、`~2`。
- 每个 voxel block 会生成一行 `setblock` 命令。
- `setblock` 行数等于模型的 `blocks.length`。

## 9. 在 Minecraft 中使用 .mcfunction

当前导出的 `.mcfunction` 适合作为 Minecraft Java Edition datapack 工作流的起点。

基本思路：

1. 创建或打开一个 Minecraft Java Edition 世界。
2. 创建 datapack。
3. 把导出的 `.mcfunction` 文件放入 datapack 的 `functions` 目录。
4. 在游戏内运行对应 function。

示例 datapack 路径大致如下：

```text
world/
└─ datapacks/
   └─ blockforge_pack/
      ├─ pack.mcmeta
      └─ data/
         └─ blockforge/
            └─ functions/
               └─ medieval-tower.mcfunction
```

游戏内运行示例：

```text
/function blockforge:medieval-tower
```

注意：v0.1 当前只导出 `.mcfunction` 文件，不会自动生成完整 datapack ZIP。完整 datapack 导出是后续计划。

## 10. 当前限制

v0.1 版本有以下限制：

- Prompt 输入不会生成新模型。
- 暂不支持材质贴图，只使用纯色材质。
- 暂不支持 `.schem` 文件导出。
- 暂不支持完整 datapack ZIP 导出。
- 大型模型暂未使用 `InstancedMesh` 优化。
- Minecraft block 映射是基础版，暂不处理朝向、门的上下半部分、水流状态等复杂 block state。

## 11. 常见问题

### 为什么我输入 prompt 后模型没有变化？

当前版本没有接入 AI 或规则生成 prompt 模型。Prompt 只用于展示交互状态。请选择 preset 查看真实模型。

### Export JSON 和 Export .mcfunction 有什么区别？

`Export JSON` 导出 BlockForge 自己的 voxel 数据结构，适合开发和二次处理。

`Export .mcfunction` 导出 Minecraft 命令文件，适合放入 Java Edition datapack 工作流。

### 为什么 .mcfunction 里的坐标都是 ~？

`~` 是 Minecraft 的相对坐标。BlockForge 默认使用相对坐标，方便你在当前位置生成建筑。

### 为什么 3D 模型和 Minecraft 里看起来可能不完全一致？

当前 3D 预览使用纯色 cube 材质，并没有 Minecraft 原版贴图、block state 或特殊方块形状。它用于快速查看 voxel 结构，不是完整 Minecraft 渲染器。

## 12. 面向开发者

核心代码位置：

```text
src/lib/voxel/
├─ types.ts              voxel 类型导出
├─ presets.ts            5 个 preset 生成器
├─ utils.ts              校验和通用工具
├─ blockStyles.ts        3D 预览颜色映射
├─ rendering.ts          渲染坐标转换
├─ minecraftBlocks.ts    Minecraft block id 映射
├─ mcfunction.ts         .mcfunction 导出
└─ exportUtils.ts        文件名工具
```

常用验证命令：

```bash
pnpm lint
pnpm test
pnpm build
```

新增或修改 preset 时，建议确认：

- blocks 数量不为空。
- 坐标不越界。
- 没有重复坐标。
- `validateVoxelModel()` 返回 `true`。
- JSON 和 `.mcfunction` 导出仍然正常。
