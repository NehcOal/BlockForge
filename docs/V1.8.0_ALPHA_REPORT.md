# BlockForge v1.8.0-alpha.1 Report

## Summary

`v1.8.0-alpha.1` prepares BlockForge as a Web Polish + In-game GUI Search /
Pagination / Filtering Alpha. It keeps the project local-first, does not add an
external AI API adapter, and does not introduce new building-generation
algorithms beyond the existing Local Rule Generator.

## Version

- Web package: `1.8.0-alpha.1`
- NeoForge Connector: `1.8.0-alpha.1`
- Fabric Connector: `1.8.0-alpha.1`
- Forge Connector: `1.8.0-alpha.1`
- Minecraft Java Edition: `1.21.1`
- Java: `21`
- Mod ID: `blockforge_connector`

## Web Polish

- Import summary now distinguishes success, warning, and error states.
- Import summary shows imported type, id/name, size, block count, palette
  count, warning count, error count, and source filename.
- Import errors now show a user-readable reason and expandable developer
  details.
- Validation report is sectioned by Model, Size, Origin, Palette, Blocks,
  Coordinates, Duplicate blocks, and Missing palette references.
- Validation issues include severity, path, message, and optional suggestion.
- Import summary and validation report are collapsible.
- Local generation copy remains Local Rule Generator / Local Prompt Rule
  Generator / 本地规则生成器.
- External AI API adapter remains planned for v2.0.
- No prompt is sent to any server.

## In-game GUI Query

NeoForge, Fabric, and Forge now expose the same Alpha GUI query behavior:

- Search input
- Source filter: All, Loose, Pack, Schematic
- Warning filter: All, With warnings, Without warnings
- Sort modes: Name, Blocks, Source
- Previous / Next pagination
- Page indicator and total blueprint count
- Source tags
- Best-effort warning badges
- Details panel source, size, block count, warning count, material status, and
  rotation controls

## Common DTOs

The loader-neutral common layer now includes:

- `BlueprintGuiQuery`
- `BlueprintSourceFilter`
- `BlueprintWarningFilter`
- `BlueprintSortMode`
- `PagedBlueprintResult`
- `BlueprintGuiQueryService`

`BlueprintSummary` now carries source type, source id, warning count, and tags.
`BlueprintListView` now carries page metadata and selected blueprint state.

## Server-side Flow

1. Client sends `BlueprintListRequestPayload` with search/filter/sort/page data.
2. Server reads the authoritative blueprint registry.
3. Server applies `BlueprintGuiQueryService`.
4. Server returns only the current page in `BlueprintListPayload`.
5. Selection still uses a separate server-validated request.
6. Builder Wand, Ghost Preview, rotation, and materials selected continue to use
   the selected blueprint state.
