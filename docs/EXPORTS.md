# BlockForge Export Formats

BlockForge v0.2 supports three export formats for the currently selected `VoxelModel`.

## JSON

The JSON export writes the complete internal `VoxelModel`.

Use it for:

- Debugging generated voxel data.
- Building custom exporters.
- Passing models to future AI or mod integrations.

## Minecraft `.mcfunction`

The `.mcfunction` export writes one `setblock` command per voxel block.

Example:

```mcfunction
setblock ~ ~ ~ minecraft:stone_bricks replace
setblock ~1 ~0 ~2 minecraft:glass replace
```

Rules:

- Commands do not include a leading `/`.
- Coordinates use the original voxel coordinates.
- Default coordinates are relative, using `~`, `~1`, `~2`, and so on.
- The number of `setblock` lines equals `model.blocks.length`.

## Minecraft Java 1.21.1 Data Pack ZIP

The Data Pack ZIP export creates a vanilla Minecraft Java Edition 1.21.1 data pack.

Target version:

- Minecraft Java Edition: `1.21.1`
- `pack_format`: `48`
- Function directory: `data/<namespace>/function`

Generated structure:

```text
pack.mcmeta
data/blockforge/function/build/<blueprint_id>.mcfunction
README.txt
```

`pack.mcmeta`:

```json
{
  "pack": {
    "pack_format": 48,
    "description": "Generated by BlockForge"
  }
}
```

Install:

1. Copy the generated zip into `.minecraft/saves/<world>/datapacks`.
2. Run `/reload` in Minecraft.
3. Run `/function blockforge:build/<blueprint_id>`.

Notes:

- Only Minecraft Java Edition is supported.
- Bedrock Edition is not supported.
- Block state details such as door half, water flow, or block orientation are intentionally minimal in this release.
- `.schem`, `.nbt`, and BlockForge Mod integration are planned separately.
