# BlockForge Export Formats

BlockForge supports several export formats for the currently selected `VoxelModel`.

## JSON

The JSON export writes the complete internal `VoxelModel`.

Use it for:

- Debugging generated voxel data.
- Building custom exporters.
- Passing models to future AI or mod integrations.

## BlockForge Blueprint v1 JSON

The Blueprint v1 export writes a stable JSON protocol for future BlockForge Mod
Connector workflows. Unlike the internal `VoxelModel`, this format is intended to
remain compatible across Web and Mod releases.

Generated file name:

```text
blockforge-<blueprint_id>.blueprint.json
```

Key rules:

- `schemaVersion` is always `1`.
- `minecraftVersion` is always `1.21.1`.
- `generator` is always `BlockForge`.
- `origin` is `{ "x": 0, "y": 0, "z": 0 }`.
- `blocks` keep raw voxel coordinates.
- `palette` maps BlockForge block names to Minecraft Java block ids.

See [Blueprint Protocol](./BLUEPRINT_PROTOCOL.md) for the full contract.

## BlockForge Blueprint v2 JSON

Blueprint v2 extends the shared protocol with Minecraft BlockState properties.

Key differences from v1:

- `schemaVersion` is `2`.
- `palette` values are objects with `name` and optional `properties`.
- `blocks` use `state` instead of `block`.
- The NeoForge Connector can apply supported string properties to the
  `defaultBlockState()`.

Example palette entry:

```json
{
  "door_lower": {
    "name": "minecraft:oak_door",
    "properties": {
      "facing": "south",
      "half": "lower",
      "hinge": "left",
      "open": "false"
    }
  }
}
```

## Minecraft `.mcfunction`

The `.mcfunction` export writes one `setblock` command per voxel block.

Example:

```mcfunction
setblock ~ ~ ~ minecraft:stone_bricks replace
setblock ~1 ~0 ~2 minecraft:glass replace
```

Rules:

- Commands do not include a leading `/`.
- Coordinates use the original voxel coordinates.
- Default coordinates are relative, using `~`, `~1`, `~2`, and so on.
- The number of `setblock` lines equals `model.blocks.length`.

## Minecraft Java 1.21.1 Data Pack ZIP

The Data Pack ZIP export creates a vanilla Minecraft Java Edition 1.21.1 data pack.

Target version:

- Minecraft Java Edition: `1.21.1`
- `pack_format`: `48`
- Function directory: `data/<namespace>/function`

Generated structure:

```text
pack.mcmeta
data/blockforge/function/build/<blueprint_id>.mcfunction
README.txt
```

`pack.mcmeta`:

```json
{
  "pack": {
    "pack_format": 48,
    "description": "Generated by BlockForge"
  }
}
```

Install:

1. Copy the generated zip into `.minecraft/saves/<world>/datapacks`.
2. Run `/reload` in Minecraft.
3. Run `/function blockforge:build/<blueprint_id>`.

Notes:

- Only Minecraft Java Edition is supported.
- Bedrock Edition is not supported.
- Block state details such as door half, water flow, or block orientation are intentionally minimal in this release.
- `.schem`, `.nbt`, and BlockForge Mod integration are planned separately.
